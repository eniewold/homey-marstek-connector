# موصل Marstek Venus

تعمل هذه التطبيق المتاح على Homey على الاتصال بنظام بطارية Marstek Venus إما عبر الشبكة المحلية أو من خلال خدمة السحابة المقدمة من Marstek، ويقوم بجمع إحصائيات البطارية. يتيح لك مراقبة حالة البطارية، مستوى الشحن، والمعلومات الأخرى ذات الصلة مباشرة من نظام Homey الذكي الخاص بك. باستخدام التدفقات، يمكنك إرسال أوامر للبطارية لتغيير الوضع (واجهة برمجة التطبيقات المحلية فقط). يوجد خوارزمية تلقائية للكشف تحاول العثور على بطاريات Marstek Venus.

## الميزات

عند اكتشاف جهاز وعندما تكون الاتصالات تعمل، سيعرض الجهاز إحصائيات متنوعة مثل:
- مستوى شحن البطارية
- الحالة (الشحن، التفريغ، الخمول)
- الطاقة المتبقية (بالكيلووات/ساعات)
- طاقة الشبكة/خارج الشبكة
- إخراج الطاقة الحالي أو المدخل (بالواط)
- درجة حرارة البطارية
- إجمالي الشحن والتفريغ (ك.و.س)

يمكنك أيضًا إرسال أوامر للبطارية لتغيير وضع التشغيل إلى 'يدوي', 'AI', 'خامل' أو 'تلقائي'. يمكن إرسال هذه الأوامر باستخدام تدفقات Homey ("ثم...").

## المتطلبات

يتطلب هذا التطبيق Homey ونظام بطارية Marstek Venus.

- عند استخدام **سائق API المحلي** يجب أن يكون كل من Homey والبطارية متصلين بنفس الشبكة مع تمكين API المحلي (انظر أدناه). يتم دعم الكشف التلقائي عن بطاريات Marstek Venus على نفس الشبكة المحلية وعندما يكون نطاق IP 192.168.x.y؛ سيتم البحث ضمن الأوكتت الأخير (y) من 1 إلى 254.
- عند استخدام **سائق السحاب** تحتاج إلى حساب Marstek سحابي نشط. أثناء الاقتران، سيطلب Homey إدخال اسم المستخدم وكلمة المرور للتحقق مع خدمة السحابة من Marstek.

### اقتران الجهاز (السحابة)

اختر جهاز "Marstek Venus (Cloud)" أثناء الاقتران، قم بتسجيل الدخول ببيانات اعتماد السحابة الخاصة بك واختر الموقع/الجهاز الذي تريد إضافته. يتم تخزين بيانات الاعتماد بأمان في متجر الأجهزة وتستخدم فقط لتحديث إحصائيات البطارية من نقاط النهاية السحابية لـ Marstek.

### اقتران الجهاز (API المحلي)

تم تعطيل API المحلي بشكل افتراضي، يحتاج إلى التمكين على نظام البطارية Marstek Venus. يمكن القيام بذلك بطريقتين:
- استخدم أداة الاختبار BLE (https://rweijnen.github.io/marstek-venus-monitor/latest/) على هاتفك الذكي (أو جهاز الكمبيوتر المحمول) قرب البطارية. قم بالاتصال واستخدم زر 'تمكين API المحلي (30000)' في علامة التبويب 'النظام'.
- تواصل مع دعم Marstek ليقوموا بتمكين API المحلي لك. قد يستغرق هذا بضعة أيام.

*يجب تمكين API المحلي لرقم المنفذ 30000 (على كل جهاز). حاليًا لا يتم دعم أرقام المنافذ الأخرى.*

## إرشادات خطوة بخطوة

1. قم بتثبيت تطبيق MarstekHomey من متجر تطبيقات Homey.
2. أضف جهاز جديد باستخدام موصل بطارية Marstek
3. اختر نوع الاتصال الذي تود القيام به:
- بالنسبة لـ API: النظام سيكتشف تلقائيًا
- بالنسبة للسحابة: أدخل بيانات اعتماد التطبيق/السحابة الخاص بك ودعه يكتشف تلقائيًا
4. اختر/تحقق من جميع الأجهزة لإضافتها واضغط على متابعة لإضافتها إلى Homey.
5. شاهد السحر يحدث حيث يتم استرجاع إحصائيات البطارية وعرضها في بطاقة الجهاز/بطاقات الأجهزة.

يمكنك الأجهزة من كل من API والسحابة. اطلع على إعدادات كل بطارية للحصول على تفاصيل إضافية.

## ملاحظات

- يستخدم هذا التطبيق ميزات "API عبر UDP" كما هو مذكور في وثائق API.
- تم تطوير التطبيق واختباره باستخدام نظام بطاريات Venus E v2.0 (إصدار البرنامج الثابت v153، وحدة الاتصال 202409090159). دعني أعرف إذا عملت أي نماذج أخرى كذلك!
- عندما لا يمكن اكتشاف الجهاز تلقائيًا، يرجى التحقق مما إذا كانت بطارية Marstek Venus قيد التشغيل ومتصلة بنفس الشبكة مثل Homey.
- تم تنفيذ دعم لعدة بطاريات Marstek Venus، ولكن بما أن لدي بطارية واحدة فقط لاختبارها، فإن بعض التجارب غير معروفة.
- يتم دعم فقط منفذ UDP 30000 حاليًا على API المحلي.
- عند ترقية التطبيق، قد يكون من الضروري إزالة أجهزة البطارية المضافة مسبقًا ثم إضافتها مجددًا.
- API السحابة من Marstek غير موثق، لذلك قد تتغير الأمور دون إشعار مسبق.
- تغييرات وضع البطارية تحتوي على محاولات إعادة تلقائية لمدة قصوى تصل إلى 5 محاولات مع فترة توقيت 15 ثانية.

## المشاكل المعروفة

- في بعض الأحيان تتوقف اتصالات UDP بعد فترة (دون أي استثناء، تحذير).
- لا يتم رد على جميع حزم UDP المرسلة من قبل البطارية (يتجاهلها بصمت).
- لا يبدو أنها تعمل جيدًا بالتزامن مع CT002 أو CT003، تتوقف البطارية عن التواصل.
- لا تأخذ بيانات السحابة منفذ الطاقة الاحتياطي بشكل صحيح بعين الاعتبار (تظهر 1 وات)
- يبدو أن استخدام جهاز السحابة يغلق التطبيق (يُسمح فقط برمز تسجيل دخول واحد بواسطة Marstek)

# استكشاف الأخطاء وإصلاحها

يحتوي API المحلي للبطارية على بعض مشاكل الاتصالات. لا تُجاب جميع رسائل UDP وهناك بعض الصراعات عندما يتم استخدام أساليب أخرى للتواصل مع البطارية في نفس الوقت. يبدو أن الاتصالات تتدهور بمرور الوقت حتى تتوقف تمامًا. مستخدمو البرامج الثابتة 154 يبلغون عن مشاكل أقل. يمكن بدء الاتصال باستخدام أداة اختبار BLE (https://rweijnen.github.io/marstek-venus-monitor/latest/) v2.0 تحت علامة التبويب "المتقدمة" باستخدام وظيفة "إعادة تعيين النظام". لاحظ أن توصيل الطاقة سيتوقف لبرهة قصيرة، وبعد ذلك سيستجيب مكدس الاتصالات مرة أخرى لجميع الرسائل.
