# Marstek Venus Connector

Это приложение Homey подключается к системе батарей Marstek Venus через локальную сеть или через облачную службу Marstek и получает статистику батареи. Оно позволяет отслеживать статус батареи, уровень заряда и другую полезную информацию прямо из вашей умной домашней системы Homey. С помощью потоков вы можете отправлять команды на батарею для изменения режима (только локальное API). Существует алгоритм автоматического обнаружения, который пытается найти ваши батареи Marstek Venus.

## ОСОБЕННОСТИ

Когда устройство обнаружено и связь работает, устройство будет отображать различные статистические данные, такие как:
- Уровень заряда батареи
- Статус (зарядка, разрядка, в режиме ожидания)
- Оставшаяся мощность (в киловатт-часах)
- Мощность от сети/вне сети
- Текущий выход или потребление мощности (Вт)
- Температура батареи
- Общие показатели заряда и разряда (кВтч)

Вы также можете отправлять команды на батарею для изменения режима работы на «Ручной», «ИИ», «Пассивный» или «Авто». Эти команды могут быть переданы с использованием потоков Homey («Тогда...»).

## ТРЕБОВАНИЯ

Это приложение требует наличия Homey и системы батарей Marstek Venus.

- При использовании **драйвера локального API** Homey и батарея должны быть подключены к одной сети с включенным локальным API (см. ниже). Поддерживается автоматическое обнаружение батарей Marstek Venus в той же локальной сети, если диапазон IP 192.168.x.y; будет осуществляться поиск в последнем октете (y) от 1 до 254.
- При использовании **облачного драйвера** вам потребуется активный облачный/приложение аккаунт Marstek. Во время подключения Homey попросит ввести имя пользователя и пароль для аутентификации с облачной службой Marstek.

### ПАРА СО С УСТРОЙСТВОМ (ОБЛАКО)

Выберите устройство “Marstek Venus (Cloud)” во время сопряжения, войдите с вашими облачными учетными данными Marstek и выберите сайт/устройство, которое вы хотите добавить. Учетные данные надежно сохраняются в хранилище устройств и используются исключительно для обновления статистики батареи с облачных точек доступа Marstek.

### ПАРА СО С УСТРОЙСТВОМ (ЛОКАЛЬНЫЙ API)

Локальный API по умолчанию отключен, его необходимо включить на системе батарей Marstek Venus. Это можно сделать двумя способами:
- Используйте BLE Test Tool (https://rweijnen.github.io/marstek-venus-monitor/latest/) на вашем смартфоне (или ноутбуке) рядом с батареей. Подключитесь и используйте кнопку 'Enable Local API (30000)' на вкладке 'System'.
- Свяжитесь с поддержкой Marstek, чтобы они включили локальное API для вас. Это может занять несколько дней.

*Локальный API должен быть включен для порта 30000 (на каждом устройстве). В настоящее время другие номера портов не поддерживаются.*

## ПОШАГОВАЯ ИНСТРУКЦИЯ

1. Установите приложение MarstekHomey из Homey App Store.
2. Добавьте новое устройство с помощью Marstek Battery Connector.
3. Выберите тип подключения:
- Для API: система автоматически обнаружит
- Для облака: введите свои облачные учетные данные и пусть оно автоматически обнаружит
4. Выберите/проверьте все устройства для добавления и нажмите «продолжить», чтобы добавить их в Homey.
5. Посмотрите, как происходит магия, когда статистика батареи извлекается и отображается в карточках устройств.

Вы можете добавлять устройства как из API, так и из облака. Смотрите настройки каждой батареи для дополнительных деталей.

## ПРИМЕЧАНИЯ

- Это приложение использует функции 'API over UDP', упомянутые в документации по API.
- Приложение разработано и протестировано с системой батареи Venus E v2.0 (прошивка v153, модуль связи 202409090159). Дайте мне знать, если какие-либо другие модели также работают!
- Если устройство не может быть автоматически обнаружено, проверьте, включена ли батарея Marstek Venus и подключена ли она к той же сети, что и Homey.
- Поддержка нескольких батарей Marstek Venus реализована, но так как у меня есть только одна батарея для тестирования, некоторые вещи неизведаны.
- В настоящее время поддерживается только порт UDP 30000 на локальном API.
- При обновлении приложения, возможно, потребуется удалить ранее добавленные устройства батареи и добавить их снова.
- API Marstek Cloud не документирован, поэтому вещи могут измениться без уведомления.
- Изменения режима батареи имеют автоматическую повтора на максимум 5 попыток с таймаутом 15 секунд.

## ИЗВЕСТНЫЕ ПРОБЛЕМЫ

- Иногда связь UDP прекращается через некоторое время (без исключения, предупреждения).
- Не все переданные пакеты UDP отвечаются батареей (она их тихо игнорирует).
- Кажется, что это плохо работает в сочетании с CT002 или CT003, батарея перестает общаться.
- Облачные данные не учитывают резервный порт питания правильно (показывает 1 Вт).
- Использование облачного устройства, похоже, приводит к выходу из приложения (разрешен только один токен входа Marstek).

# УСТРАНЕНИЕ НЕПОЛАДОК

Локальный API батареи имеет некоторые проблемы с коммуникацией. Не на все сообщения UDP отвечаются, и кажется, что есть некоторые конфликты при использовании других методов связи с батареей одновременно. Связь со временем ухудшается, пока полностью не прекращается. Пользователи с прошивкой 154 сообщают о меньших проблемах. Связь можно запустить заново, используя BLE Test Tool (https://rweijnen.github.io/marstek-venus-monitor/latest/) v2.0 на вкладке 'Advances' с использованием функции 'System Reset'. Обратите внимание, что подача энергии будет нарушена на короткий момент, после чего стек связи снова ответит на все сообщения.
